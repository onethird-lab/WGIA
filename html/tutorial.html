Its framework facilitates high-speed pairwise interaction analysis, enabling extensive exploration of genome-wide interactions in the era of big data in molecular biology.

Supported file types

Support text files: .txt, .csv, .tsv. If the file is too large (>10M), please upload a compressed file.(.rar, .zip)

Input

data matrix

Design matrix

The first column of the design matrix contains the sample names. The second and third columns denote labels for the case and control groups. For samples in the case group, the case column carries the label '1' and the control column, '0'. Conversely, for samples in the control group, the case column holds '0' while the control column holds '1'.

To expedite file uploads, WGIA exclusively accepts zip files containing the genome data matrix and design matrix, respectively.

Output

WGIA offers an example report page for users to review, providing access to all results, including visualizations, interaction tables, and download options.

Why can't I upload my file?



File upload issues may be related to the user's internet speed or file size. The WGIA online interface only supports .txt files separated by tabs or .csv files separated by commas. If the file size is too large, please compress the single text file into a separate archive before uploading.

Why is there no report after the analysis?



In that case, it may be due to the following reasons: First, the data uploaded by the user did not meet the required criteria, preventing the analysis from running correctly and resulting in no output files. Second, there may have been an unexpected interruption in the network server during the analysis. In the event of missing reports, verifying the uploaded files and carefully reviewing the user manual for proper procedure is advisable. Additionally, please contact the website administrators for further assistance.

Do I need to log in to the analysis?



WGIA does not require users to log in to perform analyses; however, each analysis job generates a unique key, which users must retain to access their historical analysis records. To better manage and view submitted analysis records, avoid losing keys, and receive email notifications at the start and end of tasks, we recommend users register and log in before using the platform.
